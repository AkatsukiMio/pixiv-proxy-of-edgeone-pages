# EdgeOne Pages 反向代理 Pixiv 图片
本项目是一个基于 EdgeOne Pages 部署的反向代理，可实现 Pixiv 图片反向代理，允许用户通过 EdgeOne Pages 的边缘节点代理访问 Pixiv 的任意图片
您可以通过访问首页输入目标图片的原 URL 或直接替换 URL 中的域名得到加速后的代理链接。
同时该项目具备可移植性和可重复性，无需改变任何内容即可进行部署

### 项目解析
```
# 首页请求
GET https://example.com
→ https://example.com/index.html
# 图片请求
GET https://example.com/img-master/img/2025/05/17/16/37/31/130502656_p0_master1200.jpg
→ https://i.pximg.net/img-master/img/2025/05/17/16/37/31/130502656_p0_master1200.jpg
```

### 部署步骤

1.  **Fork 本项目**
    点击本仓库右上角的 **Fork** 按钮，将此项目复刻到您自己的 GitHub 账户下

2.  **登录 EdgeOne 控制台**
    访问 [腾讯云 EdgeOne 控制台](https://console.tencentcloud.com/edgeone/makers)

3.  **新建项目**
    - 在左侧菜单中，选择 **站点服务** > **Makers**
    - 点击 **导入 Git 仓库** 按钮

4.  **关联 GitHub 仓库**
    - 在部署方式中，选择 **关联 Git 仓库进行部署**
    - 点击 **授权 Pages**，在弹出的窗口中授权腾讯云访问您的 GitHub 仓库
    - 授权成功后，选择您刚刚 Fork 的仓库 `pixiv-proxy-of-edgeone-pages`

5.  **配置构建选项**
    请准确按照以下配置填写：
    - **项目名称**：自定义
    - **加速区域**：选择 **`全球可用区（不含中国大陆）`**(严禁修改)
    - **生产分支**：选择 **`main`**
    - **框架预设**：选择 **`Other`**
    - **根目录**：保持默认的 **`./`** 即可
    - **输出目录**：**`static`** (请勿修改)
    - **构建命令**：**`留空`**
    - **安装命令**：**`留空`**

6.  **保存并部署**
    - 点击 **保存并部署**。
    - EdgeOne Pages 会开始拉取您的代码并进行部署。这个过程通常在几分钟内完成

7.  **访问您的代理**
    - 部署成功后，平台会自动生成一个 `<yourproject>.edgeonepage.com` 的免费域名，也可以绑定自己的域名否则中国大陆访问需要鉴权
    - 点击该域名，即可访问您的反向代理服务
